# 電子抽獎系統

這是一個前端電子抽獎介面，可將參與者名單與獎項透過 Excel 檔案分別匯入，並依標籤限制抽獎範圍。每次抽獎會自動排除已得獎者，並保留完整的歷程紀錄。

## Excel 匯入格式
使用任何支援 `.xlsx` 或 `.xls` 的軟體（Excel、Google 試算表等）製作檔案後匯出。欄位名稱可使用下列其中一個對應文字。

### 參與者名單
| 必要欄位 | 可用欄位名稱 | 說明 |
| --- | --- | --- |
| 姓名 | `姓名`、`Name`、`參與者`、`員工`、`成員` | 參與者顯示名稱 |
| 標籤 | `標籤`、`Tags`、`Tag`、`屬性`、`分類` | 以逗號、空白或頓號分隔的多個標籤（例如 `5年以上, 主管`） |

> 每一列代表一位參與者，可填入多個標籤以利後續篩選。

### 獎項設定
| 必要欄位 | 可用欄位名稱 | 說明 |
| --- | --- | --- |
| 獎項名稱 | `獎項`、`Prize`、`獎品`、`名稱` | 顯示於抽獎結果的獎項名稱 |
| 數量 | `數量`、`Quantity`、`份數` |（選填）若缺漏或非正整數則預設為 1 |
| 標籤 | `標籤`、`Tags`、`Tag`、`限制`、`分類` |（選填）限制可參與此獎項抽獎的標籤條件 |
| 照片連結 | `圖片`、`照片`、`Image`、`ImageUrl`、`Photo`、`Picture`、`圖檔`、`連結` |（選填）提供可公開存取的圖片網址以顯示獎項縮圖 |

> 若獎項指定了標籤，抽獎時會自動限制符合該標籤的參與者；仍可額外勾選其他篩選條件以縮小範圍。
>
> 若提供「照片連結」，頁面會於獎項卡片中顯示縮圖；若未提供則僅顯示文字資訊。

## 操作流程
1. 於頁面上「匯入參與者名單」選擇參與者 Excel 檔案。
2. 於「匯入獎項設定」選擇獎項 Excel 檔案。
3. 在「抽獎設定」中點選要抽出的獎項卡片；下方會顯示該獎項的剩餘數量與限定標籤。
4. 依需求勾選額外的參與者標籤，或點擊「清除篩選」快速重置。
5. 按下「開始抽獎」，系統會從符合條件且尚未得獎的名單中隨機選出中獎者，並將其自動移出後續抽獎。
6. 抽獎結果與條件會記錄在「抽獎紀錄」清單中，必要時可使用「重設抽獎紀錄」重新開始。

## 功能特點
- 支援參與者與獎項分開匯入，標籤可用於年資、職位等條件篩選。
- 獎項可設定多份數量，抽獎後自動更新剩餘份數並移除已抽完的獎項。
- 可為獎項匯入圖片連結，抽獎卡片會自動顯示縮圖，提升活動視覺效果。
- 已中獎者會自動排除，避免重複得獎。
- 抽獎歷程包含條件標籤與時間戳記，方便追蹤每一次抽獎紀錄。

## 前端相依套件
- [Bootstrap 5.3](https://getbootstrap.com/)：版型與元件樣式。
- [SheetJS (xlsx)](https://sheetjs.com/)：讀取與解析 Excel 檔案。

若要將本地變更推送到 GitHub，可依以下流程：
```bash
git status -sb
# 確認要提交的檔案

git add .
# 加入暫存區

git commit -m "Describe your changes"
# 建立提交

git push origin <branch-name>
# 推送到遠端分支
```
